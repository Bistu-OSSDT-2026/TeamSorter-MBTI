# 贡献说明

本项目用于开源实验课的小组协作实践，欢迎按统一流程参与贡献。

## 基本流程

1. 查看已有 Issue，避免重复开发
2. 若没有对应任务，先创建 Issue 说明目标、结果和负责人
3. 认领 Issue 后，从 `master` 创建开发分支
4. 在本地完成开发、自测和必要说明更新
5. 提交 Pull Request，并关联对应 Issue
6. 由其他组员进行 Review、验证和反馈
7. 修改通过后合并到主分支

## 分支命名建议

- `feature/xxx`
- `fix/xxx`
- `docs/xxx`
- `test/xxx`

## Commit 信息建议

- `feat: 新增 MBTI 测评页面`
- `fix: 修复岗位匹配接口错误`
- `docs: 更新 README 运行说明`
- `test: 补充接口验证步骤`

## Pull Request 检查项

提交 PR 前建议确认：

- 功能是否对应某个 Issue
- 是否能够正常运行
- 是否影响核心流程
- 是否更新了必要文档
- 是否说明了已知限制

## Review 重点

- 是否完成任务目标
- 是否引入明显 Bug
- 是否破坏前后端接口
- 是否影响现有页面使用
- 是否可以顺利运行和验证

## 课程实践建议

- 每位成员至少认领并完成一个可追踪任务
- 每个功能修改尽量通过 PR 合并，不直接在主分支提交
- 至少保留一次真实 Bug 的发现、修复、验证记录
- 发布版本前集中完成测试、文档补充和问题清理
